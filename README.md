AdventureWorks Enterprise Star Schema Data Modeling (SQL & Power BI)

##  Project Overview
This advanced portfolio project transitions from basic flat-file data manipulation to multi-table Enterprise Relational Data Modeling. It implements a fully normalized Star Schema data warehouse architecture inside MS SQL Server and maps dimensional parameters inside Power BI.

##  Technical Stack
   *Database Engine:* Microsoft SQL Server (T-SQL Data Warehouse)
   *Data Architecture:* Star Schema Optimization (1-to-Many Relationships)
   *Modeling Front-End:* Power BI Desktop
  *Analytical Expressions:* Advanced DAX (Context Switching & Role-Playing Dimensions)

##  Key Technical Milestones
   *Data Warehouse Restoration:* Restored a production-grade enterprise backup (.bak) file using secure system paths and administrative database controls inside SSMS.
   *Role-Playing Dimensions:* Isolated multi-key structural path traps within the calendar schema (OrderDateKey, ShipDateKey) by deploying active and inactive validation routes.
   *Advanced DAX Optimization:* Authored a standalone USERELATIONSHIP measure wrapped inside a CALCULATE expression to dynamically bypass default active paths and track supply chain logistics pipelines.
   *Cross-Dimensional Filtering:* Configured responsive multi-table dashboard slicers linking customer demographics data (DimCustomer) directly to core transaction rows (FactInternetSales).
